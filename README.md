# 🎭 数字分身 - AstrBot 人格克隆插件

> ✨ 让 AI 学会群友的说话风格，打造独一无二的数字分身！

## 📖 简介

**数字分身** 是一个 AstrBot 插件，通过分析群成员的历史消息，提取其性格特征、说话风格、口头禅等元素，生成独特的「人格画像」。随后，AI 可以基于这个画像，模拟该成员的说话方式进行回复。

简单来说：**让机器人学会像你的群友那样说话！**

## ✨ 功能特性

- 🔍 **智能分析** - 自动收集群历史消息，使用 LLM 分析生成人格画像
- 🎨 **风格模仿** - AI 基于画像模拟目标用户的说话方式、表情使用习惯
- 💬 **对话记忆** - 每个分身拥有独立的对话历史，支持自动压缩
- 🔔 **持续唤醒** - 激活后可持续响应，无需每次手动触发
- 🛡️ **合规机制** - 克隆他人需对方确认，保护用户隐私
- 🌐 **全平台支持** - 支持 QQ、Telegram、Discord、微信等所有 AstrBot 接入的平台
- ⚙️ **灵活配置** - 可自定义 LLM 提供商、时间范围、参数等

## 🌟 优势

| 特性 | 说明 |

## 📦 安装

### 前置依赖

**必须先安装 `astrbot_plugin_message_recorder` 插件**，本插件依赖它获取历史消息。

1. 在 AstrBot 插件市场搜索「消息记录器」或 `astrbot_plugin_message_recorder`，安装
2. 确保消息记录器插件正常运行，开始记录群消息

### 安装本插件

**方式一：通过 AstrBot 插件商店**

在 AstrBot 管理面板的插件市场搜索「数字分身」或 `digital_member`，一键安装。

**方式二：手动安装**

```bash
# 克隆仓库到 AstrBot 的 addons 目录
cd astrbot/addons/plugins
git clone https://github.com/Leafliber/astrbot_plugin_digital_member.git
```

重启 AstrBot 即可生效。

## 🎮 使用方法

### 基础命令

所有命令以 `/mb` 为前缀：

| 命令 | 功能 | 权限 |
|------|------|------|
| `/mb 分析 @用户 [时间范围]` | 分析目标用户生成画像 | 所有人 |
| `/mb 确认` | 确认他人发起的克隆请求 | 被克隆者 |
| `/mb 询问 [内容]` | 与已激活的分身对话 | 所有人 |
| `/mb 唤醒 @用户` | 激活分身进入持续响应模式 | 所有人 |
| `/mb 休眠 @用户` | 让分身进入休眠状态 | 所有人 |
| `/mb 画像 @用户` | 查看分身的画像详情 | 所有人 |
| `/mb 列表` | 查看本群所有分身 | 所有人 |
| `/mb 删除 @用户` | 删除指定分身 | 管理员/被克隆者/发起者 |
| `/mb 清空` | 清空本群所有分身 | 管理员 |

### 详细用法

#### 1️⃣ 分析并创建分身

```
/mb 分析 @张三 30天
```

- `@张三`：要克隆的目标用户（必须 @ 提及）
- `30天`：分析的时间范围（可选：`7天`、`30天`、`90天`、`全部`）

**合规说明**：
- 克隆自己：直接创建分身
- 克隆他人：需要对方在群内发送 `/mb 确认` 才能完成

#### 2️⃣ 确认克隆请求

当有人要克隆你时，你会收到提示。若同意，发送：

```
/mb 确认
```

#### 3️⃣ 与分身对话

**单次对话**：
```
/mb 询问 今天天气怎么样？
```

分身会基于画像以目标用户的风格回复。

**持续对话模式**：
```
/mb 唤醒 @张三
```

激活后，分身会自动响应群内消息（无需每次输入命令）。

**休眠**：
```
/mb 休眠 @张三
```

让分身停止自动响应。

#### 4️⃣ 查看画像

```
/mb 画像 @张三
```

显示分身的性格特征、说话风格、口头禅、兴趣爱好等。

#### 5️⃣ 管理分身

**查看列表**：
```
/mb 列表
```

**删除分身**：
```
/mb 删除 @张三
```

仅以下人员可执行：
- Bot 管理员
- 被克隆者本人（张三）
- 发起该克隆的人

**清空所有**（管理员专用）：
```
/mb 清空
```

## ⚙️ 配置说明

在 AstrBot 管理面板的插件配置页面可设置以下参数：

### 基础配置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `default_time_range` | 一键克隆默认时间范围 | `30天` |
| `analyze_provider_id` | 分析画像使用的 LLM 提供商 ID | 空（使用默认） |

### 消息查询设置 (`query`)

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `max_count` | 查询上限（0=不限制） | `0` |
| `fetch_context` | 是否获取上下文消息 | `false` |
| `context_before` | 获取消息前多少条 | `3` |
| `context_after` | 获取消息后多少条 | `3` |

### 画像分析设置 (`analysis`)

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `batch_size` | 每批次分析消息数 | `100` |
| `mode` | 分析模式 | `batch_summarize` |
| `batch_delay_ms` | 批次间延迟（毫秒） | `1000` |

**分析模式说明**：
- `single`: 取样本一次性分析，可能因上下文过长导致失败
- `batch_summarize`: 分批次分析后汇总，更稳定可靠

### 对话管理设置 (`conversation`)

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `max_history_turns` | 每个分身最大对话轮数 | `20` |
| `compress_threshold` | 自动压缩触发阈值 | `15` |
| `session_timeout_minutes` | 持续唤醒无活动休眠时间 | `5分钟` |

### LLM 提供商配置

插件使用 AstrBot 的 LLM 接口进行画像分析和对话生成。

- **默认**：留空 `analyze_provider_id`，使用当前会话的默认 LLM
- **指定**：输入提供商 ID，如 `openai_chat`、`deepseek_chat` 等

## 🔧 工作原理

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ 消息记录器   │ --> │ 画像分析器   │ --> │ 对话管理器   │
│ ( recorder ) │     │ (LLM分析)    │     │ (历史压缩)   │
└─────────────┘     └─────────────┘     └─────────────┘
       ↓                   ↓                   ↓
   群历史消息          人格画像JSON         对话历史
```

1. **消息获取**：通过 `astrbot_plugin_message_recorder` 插件查询历史消息
2. **画像分析**：LLM 分析消息提取性格、风格、口头禅等特征
3. **对话生成**：基于画像构建 system prompt，生成风格化回复
4. **历史压缩**：对话超过阈值自动压缩为摘要，保留关键信息
